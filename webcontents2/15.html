Configure the Container as a System Start-Up Service and Mount Volumes Persistently

Image: admin034/monitor:latest

  1. Create the container as alonewolf user.
  2. Run the container by using the image.
  3. Create the container as a system start-up service:
    • The service should automatically start after a system reboot without any human intervention.
  4. Name the system service: container-pdf_gen.
  5. Mount volumes persistently:
    • Mount the local directory /opt/in to the container’s /opt/receiver directory.
    • Mount the local directory /opt/out to the container’s /opt/sender directory.
  6. Service Functionality:
    • During the service's operation, any file created or stored in /opt/in should automatically be converted into a PDF and saved in the /opt/sender directory.
